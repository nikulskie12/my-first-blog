                                <h1 id="if-youre-doing-the-tutorial-at-home">If you're doing the tutorial at home<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#if-youre-doing-the-tutorial-at-home" aria-label="Anchor link for: if youre doing the tutorial at home" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h1>
<p>If you're doing the tutorial at home, not at one of the <a href="https://djangogirls.org/events/" target="_blank">Django Girls events</a>, you can completely skip this chapter now and go straight to the <a href="https://tutorial.djangogirls.org/en/how_the_internet_works/">How the Internet works</a> chapter.</p>
<p>This is because we cover installing things as they are needed in the tutorial -- this is just an additional page that gathers all of the installation instructions in one place (which is useful for some workshop formats). You can choose to install everything that is on this page right now if you wish. But if you want to start learning things before installing a bunch of stuff on your computer, skip this chapter and we will explain the installation parts to you later on, as they are needed.</p>
<p>Good luck!</p>
<h1 id="if-youre-attending-a-workshop">If you're attending a workshop<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#if-youre-attending-a-workshop" aria-label="Anchor link for: if youre attending a workshop" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h1>
<p>If you are attending one of the <a href="https://djangogirls.org/events/" target="_blank">Django Girls events</a>:</p>
<ul>
<li>Your workshop may have an "installation party" before the main workshop. If you are at an installation party, this page is for you! Follow the instructions here to get everything you need for the workshop installed, with the help of the coaches if needed. Then at the main workshop, you'll be able to skip installation instructions you'll encounter in the main tutorial when you get to them.</li>
<li>The organizers of your workshop may have asked you to try at home to install everything on your computer before the workshop starts. If you have been asked to do that, this page is for you! Follow the instructions here, as best you can. Then at the main workshop, when you get to an installation step in the main tutorial, if you were not able to get that piece installed you can get help from your coach.</li>
<li>If your workshop does not have an installation party (or you couldn't attend), and if the organizers didn't ask you to try to install everything before you arrived, skip this page and go straight to the <a href="https://tutorial.djangogirls.org/en/how_the_internet_works/">How the Internet works</a> chapter. You'll be installing everything you need as you work through the tutorial.</li>
</ul>
<h1 id="installation">Installation<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#installation" aria-label="Anchor link for: installation" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h1>
<p>In this tutorial you will be building a blog. In order to do that, as you go through the tutorial you'll be instructed on how to install various software on your computer and set up some online accounts as they are needed. This page gathers all of the installation and sign-up instructions in one place (which is useful for some workshop formats).</p>
<sec data-title="Chromebook setup (if you&#39;re using one)" data-id="chromebook_setup" data-collapse="true"><div class="panel panel-default"><div class="panel-heading"><b>Chromebook setup (if you're using one)<a class="pull-right section atTitle btn btn-default" target="chromebook_setup"><span class="fa fa-angle-down"></span></a></b></div><div class="panel-collapse collapse" id="chromebook_setup" aria-expanded="false" style="height: 2252.61px;"><div class="panel-body">
<p>You can <a href="http://tutorial.djangogirls.org/en/installation/#install-python" target="_blank">skip right over this section</a> if you're not using a Chromebook. If you
are, your installation experience will be a little different. You can ignore the
rest of the installation instructions.</p>
<h3 id="cloud-ide-paizacloud-cloud-ide-aws-cloud9">Cloud IDE (PaizaCloud Cloud IDE, AWS Cloud9)<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#cloud-ide-paizacloud-cloud-ide-aws-cloud9" aria-label="Anchor link for: cloud ide paizacloud cloud ide aws cloud9" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h3>
<p>Cloud IDE is a tool that gives you a code editor and access to a computer running
on the Internet where you can install, write, and run the software. For the duration
of the tutorial, cloud IDE will act as your <em>local machine</em>. You'll still be
running commands in a terminal interface just like your classmates on OS X,
Ubuntu, or Windows, but your terminal will be connected to a computer running
somewhere else that cloud IDE sets up for you.
Here is the instructions for cloud IDEs (PaizaCloud Cloud IDE, AWS Cloud9).
You can choose one of the cloud IDEs, and follow the instruction of the cloud IDE.</p>
<h4 id="paizacloud-cloud-ide">PaizaCloud Cloud IDE<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#paizacloud-cloud-ide" aria-label="Anchor link for: paizacloud cloud ide" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h4>
<ol>
<li>Go to <a href="https://paiza.cloud/" target="_blank">PaizaCloud Cloud IDE</a></li>
<li>Sign up for an account</li>
<li>Click <em>New Server</em> and choose the Django app</li>
<li>Click Terminal button(on the left side of the window)</li>
</ol>
<p>Now you should see an interface with a sidebar, buttons at the left.
Click "Terminal" button to open terminal window with prompt like this:</p>
<p></p><p class="code-label">Terminal</p><p></p>
<pre><code>$
</code></pre><p>The terminal on the PaizaCloud Cloud IDE is prepared for your instructions.
You can resize or maximize that window to make it a bit bigger.</p>
<h4 id="aws-cloud9">AWS Cloud9<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#aws-cloud9" aria-label="Anchor link for: aws cloud9" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h4>
<p>Currently Cloud 9 requires you to sign up with AWS and enter credit card
information.</p>
<ol>
<li>Install Cloud 9 from the <a href="https://chrome.google.com/webstore/detail/cloud9/nbdmccoknlfggadpfkmcpnamfnbkmkcp" target="_blank">Chrome web store</a></li>
<li>Go to <a href="https://c9.io/" target="_blank">c9.io</a> and click <em>Get started with AWS Cloud9</em></li>
<li>Sign up for an AWS account (requires credit card information, but you can
use it for free)</li>
<li>In the AWS Dashboard, enter <em>Cloud9</em> in the search bar and click it</li>
<li>In the Cloud 9 dashboard, click <em>Create environment</em></li>
<li>Name it <em>django-girls</em></li>
<li>While configuring settings, select <em>Create a new instance for environment
(EC2)</em> for "Environment Type" and the <em>t2.micro</em> "Instance type" (it should
say "Free-tier eligible."). The default cost-saving setting is fine and you
can keep the other defaults.</li>
<li>Click <em>Next step</em></li>
<li>Click <em>Create environment</em></li>
</ol>
<p>Now you should see an interface with a sidebar, a big main window with some
text, and a small window at the bottom that looks something like this:</p>
<p></p><p class="code-label">bash</p><p></p>
<pre><code>yourusername:~/workspace $
</code></pre><p>This bottom area is your terminal. You can use the terminal to send instructions
to the remote Cloud 9 computer. You can resize that window to make it a bit
bigger.</p>
<h3 id="virtual-environment">Virtual Environment<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#virtual-environment" aria-label="Anchor link for: virtual environment" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h3>
<p>A virtual environment (also called a virtualenv) is like a private box we can
stuff useful computer code into for a project we're working on. We use them to
keep the various bits of code we want for our various projects separate so
things don't get mixed up between projects.</p>
<p>Run:</p>
<p></p><p class="code-label">Cloud 9</p><p></p>
<pre><code>mkdir djangogirls
cd djangogirls
python3.6 -mvenv myvenv
source myvenv/bin/activate
pip install django~=2.2.4
</code></pre><p>(note that on the last line we use a tilde followed by an equal sign: <code>~=</code>).</p>
<h3 id="github">GitHub<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#github" aria-label="Anchor link for: github" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h3>
<p>Make a <a href="https://github.com/" target="_blank">GitHub</a> account.</p>
<h3 id="pythonanywhere">PythonAnywhere<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#pythonanywhere" aria-label="Anchor link for: pythonanywhere" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h3>
<p>The Django Girls tutorial includes a section on what is called Deployment,
which is the process of taking the code that powers your new web application
and moving it to a publicly accessible computer (called a server) so other
people can see your work.</p>
<p>This part is a little odd when doing the tutorial on a Chromebook since we're
already using a computer that is on the Internet (as opposed to, say, a laptop).
However, it's still useful, as we can think of our Cloud 9 workspace as a place
for our "in progress" work and Python Anywhere as a place to show off our stuff
as it becomes more complete.</p>
<p>Thus, sign up for a new Python Anywhere account at
<a href="https://www.pythonanywhere.com/" target="_blank">www.pythonanywhere.com</a>.</p>

<h2 id="why-are-we-installing-a-code-editor">Why are we installing a code editor?<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#why-are-we-installing-a-code-editor" aria-label="Anchor link for: why are we installing a code editor" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h2>
<p>You might be wondering why we are installing this special code editor software, rather than using something like Word or Notepad.</p>
<p>The first reason is that code needs to be <strong>plain text</strong>, and the problem with programs like Word and Textedit is that they don't actually produce plain text, they produce rich text (with fonts and formatting), using custom formats like <a href="https://en.wikipedia.org/wiki/Rich_Text_Format" target="_blank">RTF (Rich Text Format)</a>.</p>
<p>The second reason is that code editors are specialized for editing code, so they can provide helpful features like highlighting code with color according to its meaning, or automatically closing quotes for you.</p>
<p>We'll see all this in action later. Soon, you'll come to think of your trusty old code editor as one of your favorite tools. :)</p>

<h2 id="virtual-environment">Virtual environment<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#virtual-environment" aria-label="Anchor link for: virtual environment" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h2>
<p>Before we install Django we will get you to install an extremely useful tool to help keep your coding environment tidy on your computer. It's possible to skip this step, but it's highly recommended. Starting with the best possible setup will save you a lot of trouble in the future!</p>
<p>So, let's create a <strong>virtual environment</strong> (also called a <em>virtualenv</em>). Virtualenv will isolate your Python/Django setup on a per-project basis. This means that any changes you make to one website won't affect any others you're also developing. Neat, right?</p>
<p>All you need to do is find a directory in which you want to create the <code>virtualenv</code>; your home directory, for example. On Windows, it might look like <code>C:\Users\Name\</code> (where <code>Name</code> is the name of your login).</p>
<blockquote>
<p><strong>NOTE:</strong> On Windows, make sure that this directory does not contain accented or special characters; if your username contains accented characters, use a different directory, for example, <code>C:\djangogirls</code>.</p>
</blockquote>
<p>For this tutorial we will be using a new directory <code>djangogirls</code> from your home directory:</p>

<pre><code>$ mkdir djangogirls
$ cd djangogirls
</code></pre><p>We will make a virtualenv called <code>myvenv</code>. The general command will be in the format:</p>
<p></p><p class="code-label">command-line</p><p></p>
<pre><code>$ python3 -m venv myvenv
</code></pre><sec data-title="Virtual environment: Windows" data-id="virtualenv_installation_windows" data-collapse="true"><div class="panel panel-default"><div class="panel-heading"><b>Virtual environment: Windows<a class="pull-right section atTitle btn btn-default" target="virtualenv_installation_windows"><span class="fa fa-angle-up"></span></a></b></div><div class="panel-collapse collapse in" id="virtualenv_installation_windows" aria-expanded="true" style=""><div class="panel-body">
<p>To create a new <code>virtualenv</code>, you need to open the command prompt and run <code>python -m venv myvenv</code>. It will look like this:</p>
<p></p><p class="code-label">command-line</p><p></p>
<pre><code>C:\Users\Name\djangogirls&gt; python -m venv myvenv
</code></pre><p>Where <code>myvenv</code> is the name of your <code>virtualenv</code>. You can use any other name, but stick to lowercase and use no spaces, accents or special characters. It is also good idea to keep the name short â€“ you'll be referencing it a lot!</p>
</div></div></div></sec>
<sec data-title="Virtual environment: Linux and OS X" data-id="virtualenv_installation_linuxosx" data-collapse="true"><div class="panel panel-default"><div class="panel-heading"><b>Virtual environment: Linux and OS X<a class="pull-right section atTitle btn btn-default" target="virtualenv_installation_linuxosx"><span class="fa fa-angle-down"></span></a></b></div><div class="panel-collapse collapse" id="virtualenv_installation_linuxosx" aria-expanded="false" style="height: 1198.41px;"><div class="panel-body">

</code></pre></blockquote>
</div></div></div></sec>
<p>You will know that you have <code>virtualenv</code> started when you see that the prompt in your console is prefixed with <code>(myvenv)</code>.</p>
<p>When working within a virtual environment, <code>python</code> will automatically refer to the correct version so you can use <code>python</code> instead of <code>python3</code>.</p>
<p>OK, we have all important dependencies in place. We can finally install Django!</p>
<h2 id="django">Installing Django <a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#django" aria-label="Anchor link for: django" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h2>
<p>Now that you have your <code>virtualenv</code> started, you can install Django.</p>
<p>Before we do that, we should make sure we have the latest version of <code>pip</code>, the software that we use to install Django:</p>
<p></p><p class="code-label">command-line</p><p></p>
<pre><code>(myvenv) ~$ python -m pip install --upgrade pip
</code></pre><h3 id="installing-packages-with-requirements">Installing packages with requirements<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#installing-packages-with-requirements" aria-label="Anchor link for: installing packages with requirements" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h3>
<p>A requirements file keeps a list of dependencies to be installed using
<code>pip install</code>:</p>
<p>First create a <code>requirements.txt</code> file inside of the <code>djangogirls/</code> folder, using the code editor that you installed earlier. You do this by opening a new file in the code editor and then saving it as <code>requirements.txt</code> in the <code>djangogirls/</code> folder. Your directory will look like this:</p>
<pre><code>djangogirls
â”œâ”€â”€ myvenv
â”‚   â””â”€â”€ ...
â””â”€â”€â”€requirements.txt
</code></pre><p>In your <code>djangogirls/requirements.txt</code> file you should add the following text:</p>
<p></p><p class="code-label">djangogirls/requirements.txt</p><p></p>
<pre><code>Django~=2.2.4
</code></pre><p>Now, run <code>pip install -r requirements.txt</code> to install Django.</p>
<p></p><p class="code-label">command-line</p><p></p>
<pre><code>(myvenv) ~$ pip install -r requirements.txt
Collecting Django~=2.2.4 (from -r requirements.txt (line 1))
  Downloading Django-2.2.4-py3-none-any.whl (7.1MB)
Installing collected packages: Django
Successfully installed Django-2.2.4
</code></pre><sec data-title="Installing Django: Windows" data-id="django_err_windows" data-collapse="true"><div class="panel panel-default"><div class="panel-heading"><b>Installing Django: Windows<a class="pull-right section atTitle btn btn-default" target="django_err_windows"><span class="fa fa-angle-down"></span></a></b></div><div class="panel-collapse collapse" id="django_err_windows" aria-expanded="false" style="height: 207px;"><div class="panel-body">
<blockquote>
<p>If you get an error when calling pip on Windows platform, please check if your project pathname contains spaces, accents or special characters (for example, <code>C:\Users\User Name\djangogirls</code>). If it does, please consider using another place without spaces, accents or special characters (suggestion: <code>C:\djangogirls</code>). Create a new virtualenv in the new directory, then delete the old one and try the above command again. (Moving the virtualenv directory won't work since virtualenv uses absolute paths.)</p>
</blockquote>
</div></div></div></sec>
<sec data-title="Installing Django: Windows 8 and Windows 10" data-id="django_err_windows8and10" data-collapse="true"><div class="panel panel-default"><div class="panel-heading"><b>Installing Django: Windows 8 and Windows 10<a class="pull-right section atTitle btn btn-default" target="django_err_windows8and10"><span class="fa fa-angle-down"></span></a></b></div><div class="panel-collapse collapse" id="django_err_windows8and10" aria-expanded="false" style="height: 174.781px;"><div class="panel-body">
<blockquote>
<p>Your command line might freeze after when you try to install Django. If this happens, instead of the above command use:</p>
<p></p><p class="code-label">command-line</p><p></p>
<pre><code>C:\Users\Name\djangogirls&gt; python -m pip install -r requirements.txt
</code></pre></blockquote>
</div></div></div></sec>
<sec data-title="Installing Django: Linux" data-id="django_err_linux" data-collapse="true"><div class="panel panel-default"><div class="panel-heading"><b>Installing Django: Linux<a class="pull-right section atTitle btn btn-default" target="django_err_linux"><span class="fa fa-angle-down"></span></a></b></div><div class="panel-collapse collapse" id="django_err_linux" aria-expanded="false" style="height: 99px;"><div class="panel-body">
<blockquote>
<p>If you get an error when calling pip on Ubuntu 12.04 please run <code>python -m pip install -U --force-reinstall pip</code> to fix the pip installation in the virtualenv.</p>
</blockquote>
</div></div></div></sec>
<p>That's it! You're now (finally) ready to create a Django application!</p>
<h1 id="git">Install Git <a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#git" aria-label="Anchor link for: git" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h1>
<p>Git is a "version control system" used by a lot of programmers. This software can track changes to files over time so that you can recall specific versions later. A bit like the "track changes" feature in word processor programs (e.g., Microsoft Word or LibreOffice Writer), but much more powerful.</p>
<h2 id="installing-git">Installing Git<a class="anchorjs-link " href="https://tutorial.djangogirls.org/en/installation/#installing-git" aria-label="Anchor link for: installing git" data-anchorjs-icon="#" style="padding-left: 0.375em;"></a></h2>
<sec data-title="Installing Git: Windows" data-id="git_install_windows" data-collapse="true"><div class="panel panel-default"><div class="panel-heading"><b>Installing Git: Windows<a class="pull-right section atTitle btn btn-default" target="git_install_windows"><span class="fa fa-angle-down"></span></a></b></div><div class="panel-collapse collapse" id="git_install_windows" aria-expanded="false" style="height: 219.188px;"><div class="panel-body">
<p>You can download Git from <a href="https://git-scm.com/" target="_blank">git-scm.com</a>. You can hit "next" on all steps except for two: in the step where it asks to choose your editor, you should pick Nano, and in the step entitled "Adjusting your PATH environment", choose "Use Git and optional Unix tools from the Windows Command Prompt" (the bottom option). Other than that, the defaults are fine. Checkout Windows-style, commit Unix-style line endings is good.</p>
<p>Do not forget to restart the command prompt or powershell after the installation finished successfully.
</p></div></div></div></sec><p></p>
<sec data-title="Installing Git: OS X" data-id="git_install_OSX" data-collapse="true"><div class="panel panel-default"><div class="panel-heading"><b>Installing Git: OS X<a class="pull-right section atTitle btn btn-default" target="git_install_OSX"><span class="fa fa-angle-down"></span></a></b></div><div class="panel-collapse collapse" id="git_install_OSX" aria-expanded="false" style="height: 126px;"><div class="panel-body">
<p>Download Git from <a href="https://git-scm.com/" target="_blank">git-scm.com</a> and follow the instructions.</p>
<blockquote>
<p><strong>Note</strong> If you are running OS X 10.6, 10.7, or 10.8, you will need to install the version of git from here: <a href="https://sourceforge.net/projects/git-osx-installer/files/git-2.3.5-intel-universal-snow-leopard.dmg/download" target="_blank">Git installer for OS X Snow Leopard</a></p>
</blockquote>
</div></div></div></sec>
<sec data-title="Installing Git: Debian or Ubuntu" data-id="git_install_debian_ubuntu" data-collapse="true"><div class="panel panel-default"><div class="panel-heading"><b>Installing Git: Debian or Ubuntu<a class="pull-right section atTitle btn btn-default" target="git_install_debian_ubuntu"><span class="fa fa-angle-down"></span></a></b></div><div class="panel-collapse collapse" id="git_install_debian_ubuntu" aria-expanded="false" style="height: 141.172px;"><div class="panel-body">

<pre><code class="lang-bash">$ sudo apt install git
</code></pre>
</div></div></div></sec>
<sec data-title="Installing Git: Fedora" data-id="git_install_fedora" data-collapse="true"><div class="panel panel-default"><div class="panel-heading"><b>Installing Git: Fedora<a class="pull-right section atTitle btn btn-default" target="git_install_fedora"><span class="fa fa-angle-down"></span></a></b></div><div class="panel-collapse collapse" id="git_install_fedora" aria-expanded="false" style="height: 141.172px;"><div class="panel-body">

</div></div></div></sec>
